# 🦋 新しい変更内容の作成

このドキュメントでは、`pnpm changeset` を使ってパッケージのバージョン管理と公開を行う方法を説明します。
具体的には、パッケージのバージョンを更新するための変更内容を記録するプロセスについて解説します。
パッケージの変更内容を作成するには、ローカル環境で次のコマンドを実行してください：

1. 新しい変更内容の追加：コマンドを実行すると、変更内容の説明を入力するプロンプトが表示されます。これで変更内容を記録できます。
    ```bash
    pnpm changeset
    ```

2. バージョンの更新と変更内容の適用：このコマンドはすべてのパッケージのバージョンを更新し、`CHANGELOG.md` ファイルを生成または更新します。
    ```bash
    pnpm changeset version
    ```
3. 公開：このコマンドは更新されたパッケージを npm に公開します。公開には `NPM_TOKEN` が GitHub Secrets に設定されている必要があります。
    ```bash
    pnpm changeset publish
    ```

詳細は[公式ドキュメント](https://github.com/changesets/changesets)を参照してください。
