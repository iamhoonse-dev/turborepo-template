This project uses [Prettier](https://prettier.io/) to keep code style consistent.
Prettier automatically formats code, helping to reduce style differences between team members.

## How to Run

Code formatting is performed using the `prettier` CLI.
Scripts for this are defined in the root directory's `package.json` file.

```json filename="package.json" {3-4}
{
  "scripts": {
    "format": "prettier --write \"**/*.{ts,tsx,md}\"",
    "format:check": "prettier --check \"**/*.{ts,tsx,md}\""
  }
}
```

## Formatting Convention Settings

This project does not use a separate Prettier config file and follows the default settings.
If you need additional settings, you can create a `.prettierrc` file or similar in the project root and define your rules.

<Callout type="info">
  For more details on configuring Prettier, see the [official Prettier documentation](https://prettier.io/docs/configuration).
</Callout>

## Integration with ESLint

In this project, [`eslint-config-prettier`](https://github.com/prettier/eslint-config-prettier) is applied to disable rules in the ESLint config that conflict with Prettier.
This setting is defined in the `configs/eslint-config/base.js` file.

```js filename="configs/eslint-config/base.js" {2,13}
import js from "@eslint/js";
import eslintConfigPrettier from "eslint-config-prettier";

// ...

/**
 * A shared ESLint configuration for the repository.
 *
 * @type {import("eslint").Linter.Config[]}
 * */
export const config = [
  js.configs.recommended,
  eslintConfigPrettier,
  // ...existing code...
];
```
