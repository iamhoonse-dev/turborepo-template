# 📝 Empty Next App

`apps/empty-next-app` 워크스페이스는 Next.js 기반의 최소 템플릿 앱입니다.

<Callout type="info">
  이 워크스페이스는 [Turborepo 템플릿](https://github.com/vercel/turborepo/tree/main/examples/basic)에 기본적으로 포함되어 있는 Next.js 앱을 기반으로 합니다. \
  **참고한 소스 코드 링크** : [Turborepo Next.js 템플릿 > apps > web](https://github.com/vercel/turborepo/tree/main/examples/basic/apps/web)
</Callout>

## 🎯 주요 목적

- Next.js 프로젝트를 빠르게 시작할 수 있는 가장 기본적인 구조를 제공합니다.
- 불필요한 코드나 의존성이 없는 "빈" 상태로, 다양한 실험이나 신규 프로젝트의 출발점으로 활용할 수 있습니다.

<Callout type="info">
  이 워크스페이스를 이용해서 새로운 Next.js 앱을 시작하기 위한 방법은 아래의 문서를 참고하세요:
  - <LocalizedLink href="/apps/add-new-app/add-next-app">`애플리케이션 > 새 앱 추가 > 새 Next 앱 추가하기`</LocalizedLink>
</Callout>

## 🌟 주요 특징

- Next.js 공식 기본 구조만 포함되어 있습니다.
- 별도의 UI 컴포넌트, 스타일, 라우트, API 등은 포함되어 있지 않습니다.
- 필요에 따라 원하는 라이브러리나 기능을 자유롭게 추가할 수 있습니다.
- ESLint, TypeScript 설정은 공통 설정을 상속하도록 구성되어 있습니다.

<Callout type="info">
  이 프로젝트에서 상속받는 ESLint, TypeScript 관련 공통 설정에 대한 보다 자세한 내용은 아래의 2 개 문서를 참고하세요:
  - <LocalizedLink href="/configs/eslint-config">`공통 설정 > ESLint Config`</LocalizedLink>
  - <LocalizedLink href="/configs/typescript-config">`공통 설정 > TypeScript Config`</LocalizedLink>
</Callout>

## 🧑‍💻 활용 예시

- 새로운 Next.js 앱을 빠르게 생성하고 싶을 때
- 실험용 샌드박스 앱이 필요할 때
- 모노레포 내에서 Next.js 환경을 테스트하거나 비교하고 싶을 때
