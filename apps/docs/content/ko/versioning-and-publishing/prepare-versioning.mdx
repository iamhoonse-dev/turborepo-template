# 🦋 새 변경 사항 작성하기

이 문서에서는 `pnpm changeset`를 사용하여 패키지의 버전을 관리하고 배포하는 방법을 안내해 드릴게요.
정확히는 패키지의 버전을 업데이트하기 위한 변경 사항을 기록하기 위한 프로세스에 대해 설명할 거예요.
패키지의 변경 사항을 작성하려면, 로컬 환경에서 다음과 같은 명령어를 실행해 주세요:

1. 새 변경 사항 추가: 명령어를 실행하면 변경 사항에 대한 설명을 작성할 수 있는 프롬프트가 나타나요. 이를 통해 변경 사항을 기록할 수 있어요.
    ```bash
    pnpm changeset
    ```

2. 버전 업데이트 및 변경 사항 적용: 이 명령어는 모든 패키지의 버전을 업데이트하고, `CHANGELOG.md` 파일을 생성하거나 업데이트해요.
    ```bash
    pnpm changeset version
    ```
3. 배포: 이 명령어는 업데이트된 패키지를 npm에 배포해요. 배포를 위해서는 `NPM_TOKEN`이 GitHub Secrets에 설정되어 있어야 해요.
    ```bash
    pnpm changeset publish
    ```

자세한 내용은 [공식 문서](https://github.com/changesets/changesets)를 참고해 주세요.
